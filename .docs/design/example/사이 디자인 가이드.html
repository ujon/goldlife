권장

비드 심볼은 항상 안테나 비드를 살린다 · 워드마크는 Pretendard 800 · 최소 여백 = 비드 높이의 50% · 모노일 땐 잉크 단색.

금지

그라데이션 각도 변경·회전 금지 · 워드마크 자간 임의 변경 금지 · 마스코트와 워드마크를 세로로 겹쳐 쌓지 않기 · 비드를 정원(완전한 원)으로 찌그러뜨리지 않기.
02

브랜드 · 무드 · 보이스

한 줄 무드: “혼자도 여럿도, 시간·돈 안에서 가볍게 떠나는 주말 큐레이터.”

무드 키워드
  • 따뜻함 · 트렌디 · 장난기
  • 가볍고 친근한 코치(Duolingo)
  • 깔끔 · 넉넉한 여백 · 큰 라운드
톤 오브 보이스
  • 친구처럼 반말, 무례하지 않게
  • 짧게 묻고 가볍게 리액션
  • 실패해도 압박 안 함
  • “ㅋㅋ” 섞인 캐주얼 카피 OK
피해야 할 것
  • 광고 리스트 같은 추천 화면
  • 정보 빽빽한 대시보드
  • 평가하는 듯한 성향 질문
  • 딱딱한 권한 요청
카피 예시
“좋아, 너 취향부터 살짝 알아볼게.” · “오늘 비 와서 실내가 편할 듯!” · “예산보다 8,000원 여유 있어 :)” · “괜찮아, 싫으면 지역 직접 골라도 돼.”
03

마스코트 “사이”

전 과정의 안내자(host). 몸 색은 코럴(따뜻함)→인디고(차분함) 그라데이션 = 시간과 돈, 사람과 사람 “사이”의 블렌드. 머리 위 안테나로 사용자의 입력을 ‘수신’합니다. CSS 기하 프리미티브로 제작 — 추후 일러스트 교체 용이.

화면별 사용 머ABap · 음성 정책

음성 사용 — 메인 서비스만

시간·예산 입력, AI 추가 질문, 추천 결과(TTS)에서 마이크 3상태 + 실시간 전사 + 자막을 쓴다. (우리가 함께 잡은 보이스 인터랙션)

음성 미사용 — 가입·온보딩

로그인/회원가입, 위치 권한, 고정 온보딩은 탭 기반. 마스코트는 코치로 등장하되 마이크는 띄우지 않는다.
상태 매핑 — 로그인/홈 idle · 시간·예산·AI질문 듣기 listening · 추천 생성 thinking · 이유 읽기 talking · 추천 완성/좋아요 happy
04

컬러

브랜드 그라데이션(코럴→바이올렛→인디고)은 AI/사이/CTA에만. 화면 대부분은 웜 뉴트럴 + 화이트 카드로 차분하게. 상황(혼자·친구·커플·가족·모임)은 같은 명도·채도의 저채도 틴트로만 구분(무지개 금지).

상황 액센트 (oklch · 동일 L/C, 색상만 변화)
Brand Gradient · linear-gradient(90deg, #FF6B5E, #B45EE8 55%, #5B6CFF) — 각도/정지점 고정. 텍스트엔 background-clip, 면엔 CTA·미터·마이크 링.

09

화면 명세

로그인 → 위치 → 온보딩 → 홈 → 상황 → 시간 → 예산 → AI 질문 → 추천 3개 → 피드백. 상단(컨텍스트)·본문(콘텐츠)·하단(주 액션) 3단 골격, 하단 CTA sticky.

10

상태 · 폴백

로딩
추천 생성 중 → 마스코트 thinking + 카드 스켈레톤. “AI 친구들이 잠깐 회의했어” 정도로 가볍게(tobl 연출은 접거나 빠르게).
빈 결과 fallback
조건 만족 후보 부족 시 → 예산 늘리기 / 이동범위 넓히기 / 집·온라인 추천 보기 액션 칩 제시. 에러처럼 보이지 않게.
권한·STT 실패
위치 거부 → 지역 직접 입력. 음성 인식 실패 → 칩/직접입력으로 자연 폴백(메인 화면). 압박 카피 금지.
모션 원칙
화면 전환 가로 슬라이드(360ms) · 카드 stagger fade-up(0.05s 간격) · 탭 시 scale 0.97 + 햅틱 · 마이크 idle→listening→processing · 예산 미터 채움(초과 시 경고색) · prefers-reduced-motion 시 모션 최소화.
11

개발 핸드오프 노트

반영 원칙
  • 토큰(컬러·radius·shadow)은 CSS 변수로 그대로 사용 — 위 :root 참고
  • 마스코트는 mascot.css + state 클래스(sai--listening 등)로 제어
  • 추천 카드는 이유 우선 슬롯 순서 고정 · 라벨/이유 텍스트는 AI JSON에서
  • 상황 세그먼트는 5종, 색상은 상황 액센트 토큰
데이터 → UI 매핑
  • situation → 상황 세그먼트 · 카드 카피 단/복수
  • availableTime → 단일/미니코스/시간표형 결과 형태
  • weatherFit → 실내/실외 배지 · 폴백 분기
  • companionConstraints.hasBaby → 인프라 배지 + 우선 필터
  • feedback.sentiment/reasons → 좋아요·별로 바 + 이유 칩
참고 프로토타입 — 음성·마스코트·투표 인터랙션은 “사이 프로토타입.html”에서 동작 확인 가능. 본 가이드의 컴포넌트·토큰과 동일 시스템입니다.
