# Handoff: 사이 (SAI) — Situation-aware AI 추천 서비스

> 디자인 핸드오프 패키지 · v0.2 · 2026-05-30
> 이 폴더 하나로 개발 착수가 가능하도록, 디자인 의도·토큰·컴포넌트·화면·인터랙션을 정리했습니다.

---

## Overview

**사이(SAI)** 는 사용자의 **시간과 총 예산** 안에서 — 혼자/친구/커플/가족/모임 누구와든 — 무엇을 하면 좋을지 추천하는 AI 기반 활동 추천 서비스입니다. 로그인 → 위치 권한 → 고정 온보딩(성향 파악) 후, 추천을 받을 때마다 상황·시간·예산을 입력받아 AI가 추가 질문 1–2개를 던지고, 외부 API를 활용해 **실행 가능한 추천 3개**를 카드로 제시합니다.

핵심 차별점: 단순 검색이 아니라 **시간·돈·위치·날씨·성향·동행 상황**을 함께 보고, AI가 추천 라벨과 “왜 너에게 맞는지” 이유를 생성합니다.

## About the Design Files

이 번들의 HTML/CSS/JS 파일은 **디자인 레퍼런스(프로토타입)** 입니다 — 의도한 룩앤필과 동작을 보여주는 것이지, 그대로 가져다 프로덕션에 넣는 코드가 아닙니다.

**개발 과제 = 이 디자인을 대상 코드베이스의 기존 환경(React/Vue/SwiftUI/네이티브 등)·패턴·라이브러리로 “재현”하는 것**입니다. 아직 환경이 없다면 프로젝트에 가장 적합한 프레임워크를 선택해 구현하세요. 디자인 토큰(CSS 변수)과 마스코트 CSS는 그대로 차용 가능하지만, 화면 구조와 상태 관리는 대상 환경의 컨벤션을 따르세요.

마스코트 “사이”는 외부 일러스트 의존 없이 **순수 CSS 도형(mascot.css)** 으로 구현되어 있어, 그대로 포팅하거나 추후 SVG/일러스트로 교체할 수 있습니다.

## Mascot “사이”

전 과정의 안내자(host). 몸 색은 코럴→인디고 그라데이션(시간·돈, 사람 “사이”의 블렌드), 머리 위 안테나로 입력을 ‘수신’.

- 구현: `mascot.css` + 루트 클래스 `sai` 에 상태 modifier 클래스 부착. 크기는 `--sai` 변수(px).
- 마크업 구조: `proto-mascot.jsx`의 `<Mascot state size>` 컴포넌트 참고.
- **상태 5종**: `sai--idle`(홈·로그인·대기) · `sai--listening`(음성 듣기) · `sai--thinking`(추천 생성·AI 분석) · `sai--talking`(이유 읽기·TTS) · `sai--happy`(추천 완성·좋아요)
- 눈 크기는 몸의 17% 비율(고정 px 없음)로 모든 크기에서 일관.

### 음성(Voice/STT/TTS) 사용 정책 — 중요
- **사용 O — 메인 서비스만**: 사용 시간 입력, 총 예산 입력, AI 추가 질문, 추천 결과 읽기(TTS). 마이크 3상태 + 실시간 전사 + 자막.
- **사용 X — 가입/온보딩**: 로그인·회원가입, 위치 권한, 고정 온보딩은 **탭 기반**. 마스코트는 코치로 등장하되 마이크는 띄우지 않음.

## Screens / Views

화면 골격은 **상단(컨텍스트) / 본문(콘텐츠) / 하단(주 액션, sticky CTA)** 3단. 모바일 우선, 단일 컬럼, `max-width 480px` 중앙 정렬. 미니 목업은 디자인 가이드 §09 참고.

### 1. 로그인 / 회원가입
- **Purpose**: 이메일+비밀번호 인증(이메일 검증 없음). 로그인 후 사용자 ID로 온보딩 상태·히스토리 조회.
- **Layout**: 중앙 정렬 단일 컬럼. 상단 로고 락업 + 캡션 → 이메일/비밀번호 필드(`.field`, radius 16) → primary 버튼(로그인) → 텍스트 링크(회원가입).
- **Components**: 입력 필드(focus 시 `--violet` 보더 + 4px 글로우), `.btn.primary`(그라데이션). 음성·마스코트 최소.

### 2. 위치 권한
- **Purpose**: 현재 위치 기반 추천. 거부 시 지역 직접 입력 fallback.
- **Layout**: 마스코트(idle) 중앙 + 헤드라인 + 설명 → 하단 [위치 허용하기] primary + [지역 직접 고르기] ghost.
- **Copy(코치 톤)**: “근처에서 재밌는 거 찾아올게! 싫으면 지역 직접 골라도 돼.” — 딱딱하지 않게.

### 3. 고정 온보딩 (4–5문항)
- **Purpose**: 활동/소비/이동 성향을 DB에 저장. 시간·예산은 묻지 않음.
- **Layout**: 진행 스테퍼 → 마스코트 말풍선 질문(`.qbubble`) → 선택지 버튼(세로 스택 칩) → 가벼운 리액션.
- **질문 축**: `activity_preferences`, `novelty_preference`, `spending_style`, `risk_tolerance`, `mobility_preference`.
- **음성 미사용**: 탭으로만 응답.

### 4. 홈 / 추천 시작
- **Layout**: 마스코트 인사 + 큰 [오늘 뭐하지? 추천받기] primary(하단 도달권) + 최근 추천 히스토리 칩.
- 온보딩 미완료 사용자는 온보딩으로 보냄.

### 5. 오늘의 상황 선택
- **Purpose**: 추천 직전에 동행 상황 선택(세션별 저장, 기본 프로필에 고정 X).
- **Layout**: 스테퍼 → 질문 “오늘 누구랑?” → 2열 그리드 세그먼트(`.segc`) 5종: 🙋혼자 · 🧑‍🤝‍🧑친구 · 💑커플 · 👨‍👩‍👧가족 · 🎉모임. 선택 시 상황 액센트 보더+틴트.

### 6. 사용 가능 시간
- **Layout**: 스테퍼 → 질문 → 칩 그리드(1시간 / 2-3시간 / 반나절 / 하루 / 이번 주말 / 직접 입력) → 하단 마이크(메인 = 음성 가능).
- 결과 형태 결정 입력: 시간이 길수록 단일활동→미니코스→시간표형.

### 7. 총 예산
- **Layout**: 칩 그리드(1만/3만/5만/10만/15만+/직접) + **1인당 자동 계산** 표시(여러 명이어도 총 예산 기준). 마이크(음성 가능).

### 8. AI 추가 질문 (1–2개)
- **Purpose**: 날씨·프로필·위치·상황에 따라 동적 생성. 시간·예산은 다시 묻지 않음.
- **Layout**: 스테퍼 → 마스코트(thinking) 말풍선 질문 → 답변 칩 2개. 예: “오늘 비 와서 실내가 편할 듯! 수다 위주 vs 같이 뭔가?”

### 9. 추천 결과 3개
- **Purpose**: 항상 **3개**. 라벨은 AI가 상황 따라 생성. 카드는 **“이유 먼저”**.
- **Layout**: 마스코트(happy) + 헤더 → 추천 카드 3장 세로 스택. 각 카드 = 추천 카드 해부(아래) 참고.

### 10. 추천 카드 상세 + 피드백
- **Layout**: 확장 카드 + 좋아요/별로예요 바(`.fbbar`) + 별로예요 이유 칩(선택).
- **CTA**: 상세/예약/구매/길찾기 외부 송객.

## 추천 카드 해부 (Recommendation Card)

슬롯 순서 **고정** (`.rec` 컴포넌트, guide §08 참고):
1. **라벨** (AI 생성, 칩) — 예: “🌧 비 오는 날 실내픽”
2. **제목** (H3)
3. **왜 이걸 골랐는지 (이유)** — `.why`, 코럴 좌측 보더. *가장 먼저, 가장 강조.*
4. **시간 계획 · 예상 비용 · 1인당 · 이동** — `.plan` 통계 행
5. **구성 아이템** (활동/식사 등) + 개별 가격·출처·송객 링크
6. **배지** — 예산 여유(good) · 실내(violet) · 도보 N분(indigo) 등
7. **CTA** — 전체 코스 보기 / 길찾기·상세

### 아기 동반 변형 (UC-002)
가족/아기 세션은 **안전·인프라가 부모 취향보다 우선**. 배지로 `유모차 OK · 수유실 · 기저귀 교체 · 주차 · 화장실 근접` 노출. 정보 부족 후보는 `정보 부족` 표기·하향 랭킹. 등산 같은 취향은 “아기 동반 가능한 완만한 자연 코스”로 재해석.

## Components

`guide.css`에 데모 스타일 존재. 클래스 = 스펙.
- **버튼** `.btn`: `primary`(그라데이션, h52, radius16) · `secondary`(보더) · `ghost` · `disabled`(`#E3DEEA`)
- **칩** `.chip`: default / `.on`(잉크 채움) / `.accent`(바이올렛 틴트) · pill · h≈40
- **상황 세그먼트** `.segc`: 아이콘+라벨, 선택 시 상황 액센트
- **입력 필드** `.field`: radius16, focus 시 바이올렛 보더+글로우
- **마이크 3상태** `.mic`: `idle`(은은한 펄스) → `listening`(수신 링 + 파형) → `processing`(스피너). **메인 서비스 전용**
- **진행 스테퍼** `.stepper`: 현재 위치 막대형 dot
- **AI 말풍선** `.qbubble`: 마스코트 + 말풍선(이름 그라데이션)
- **피드백 바** `.fbbar`: 좋아요(good)/별로예요(coral)
- **배지** `.badge`: `g`(good) · `a`(coral) · `b`(indigo) · `w`(violet)

## Interactions & Behavior

- 화면 전환: 가로 슬라이드 360ms `cubic-bezier(.4,0,.2,1)`. (프로토타입은 Web Animations API로 구현 — `app.jsx`의 `Screen` 참고)
- 카드 등장: stagger fade-up, 0.05s 간격. 탭 시 scale 0.97 + 햅틱.
- 예산 미터: 합계가 게이지에 차오름. 초과 시 경고색(`--warn`).
- 마이크: idle→listening(수신 링/파형)→processing(스피너). 실시간 전사 텍스트 표시(잘못 들으면 즉시 수정).
- TTS 재생 중: 스피커 인디케이터 + 자막 동시 노출.
- `prefers-reduced-motion`: 모션 최소화.

### 상태 · 폴백
- **로딩**: 마스코트 thinking + 카드 스켈레톤. tobl 페르소나 토론은 선택 연출(접거나 빠르게, “AI 친구들이 잠깐 회의했어” 정도).
- **빈 결과**: 예산 늘리기 / 이동범위 넓히기 / 집·온라인 추천 보기 액션 칩(에러처럼 보이지 않게).
- **권한·STT 실패**: 위치 거부 → 지역 직접 입력. 음성 실패 → 칩/직접입력 폴백. 압박 카피 금지.

## State Management

- **Auth**: 로그인 상태, userId
- **Profile**: 온보딩 완료 여부 + 성향 5종(`activity_preferences`, `novelty_preference`, `spending_style`, `risk_tolerance`, `mobility_preference`) + `recent_location`
- **Recommendation session(추천마다 신규)**: `situation`, `availableTime`, `budgetTotal`, `location`, `weather_snapshot`, `dynamicQuestions/Answers`, `companionConstraints`(예: `hasBaby`, `strollerRequired`)
- **Results**: 추천 카드 3개(label, title, reason, resultType, estimatedCost/Duration, weatherFit, companionFit, outboundUrl)
- **Feedback**: sentiment(좋아요/별로예요) + reasons[]

### 데이터 → UI 매핑
- `situation` → 상황 세그먼트 선택 · 카드 카피 단/복수형
- `availableTime` → 결과 형태(단일활동 / 미니코스 / 시간표형)
- `budgetTotal` → 칩 선택 + 1인당 계산 + 예산 미터
- `weatherFit` → 실내/실외 배지 · 폴백 분기
- `companionConstraints.hasBaby` → 인프라 배지 + 우선 필터(취향보다 우선)
- `feedback.sentiment / reasons` → 좋아요·별로 바 + 이유 칩

데이터 모델 초안·API 전략·유스케이스는 원본 PRD(`PRD.md`) §5–7 참고.
