# PRD — 사이 (SAI)

> **OBA Weekend-thon S1 빌드 직행 PRD** · v2.0 · 2026-05-30 *(1인/2인 모드 반영)*
> 폰을 둘 **사이**에 두고(혹은 혼자), AI가 나·우리에게 맞는 나들이 코스를 생성형 카드로 뽑아준다.

---

## 0. 한 줄 정의

> "이번 주말, 15만원으로 뭐하지?" — **혼자든 둘이든** 시간·예산·MBTI를 **음성**으로 말하면, AI 페르소나들이 토론·투표해서 **나(혹은 우리)에게 맞는 나들이 코스(액티비티 + 맛집)를 생성형 카드로** 제안하고, 카드를 탭하면 실제 예약 페이지로 보낸다.

- **타깃**: 주말마다 "뭐하지"로 고민하는 20–30대 — ① **커플**(연인·동성 커플) ② **혼자** 나들이·혼놀하는 1인. 두 모드 모두 지원.
- **폼팩터**: 모바일 웹. **2인** = 폰을 둘 사이에 두고 번갈아 / **1인** = 혼자 음성·탭으로 조작
- **데모 타이틀**: 사이 (SAI) — "둘 사이 + AI" *(혼자 쓸 땐 "내 곁의 AI")*

## 2. 핵심 UX 플로우 (화면별)

```
[S0 모드 선택] "오늘 누구랑 놀까?" → [🙋 혼자] / [💑 둘이서]
   - 음성("혼자" / "여친이랑")로도 선택 → mode = solo | duo, party 결정
   - duo 선택 시 "폰을 둘 사이에 놓아주세요" 안내 노출
   ↓ (탭 또는 음성 "시작")
[S1 음성 입력] 🎙️ "이번 주말 얼마로 뭐하고 놀까?"
   - 사용자 발화 → STT → EXAONE NLU → {날짜, 예산, 인원}
   - 인원 미언급 시 S0 선택값 사용 / 발화에 인원 있으면 S0 보정
   - 못 들으면 텍스트 폴백 입력 칩 노출
   ↓
[S2 MBTI 입력] solo="네 MBTI는?" / duo="둘의 MBTI는?"
   - 음성("ENTP" 또는 "ENTP, ISFJ") 또는 4x4 MBTI 그리드 탭
   - 슬롯 수 = 인원수 (1인=1슬롯, 2인=2슬롯)
   ↓
[S3 키워드 생성] solo="네 성향 + 요즘 트렌드 키워드야" / duo="두 사람 성향 + 요즘 트렌드 키워드야" → "끌리는 거 골라봐"
   - GenRank 시드 + EXAONE 페르소나가 키워드 6~8개 카드 생성
   - 탭 선택(다중 가능). duo는 번갈아 탭
   ↓
[S4 페르소나 투표] duo="AI 친구들이 너희 보고 토론했어 👀" / solo="AI 친구들이 너 분석했어 👀"
   - tobl council 멤버 = (인원수만큼의 MBTI 페르소나) + 트렌드 페르소나
     · duo: ENTP 페르소나 vs ISFJ 페르소나 + 트렌드 페르소나
     · solo: 내 ENTP 페르소나 + 트렌드 페르소나 + 취향 대변 페르소나
   - 선택 키워드로 코스 후보를 놓고 찬반·투표 → 결과 시각화
   ↓
[S5 코스 카드] solo="너를 위한 코스야" / duo="너희를 위한 코스야"
   - 병렬 호출 결과로 코스 카드 생성:
     · 🎯 액티비티 (마이리얼트립 투어티켓)
     · 🍽️ 맛집 (API Fuse·캐치테이블)
     · ☀️ 날씨 배지 (API Fuse·기상청) → 실내/실외 자동 필터
     · 📍 동선·거리 (API Fuse·카카오맵)
   - 예산 합계 표시 (15만원 내 맞춤)
   ↓
[S6 송객] 카드 탭 → 해당 상품 상세/예약 페이지로 딥링크 이동
```

**생성형 UI 원칙**: 카드는 LLM이 반환한 JSON 스키마로 동적 렌더(고정 템플릿 X). 멀티턴 동안 이전 선택이 다음 카드에 반영(컨텍스트 반응).

## 3. 아키텍처 / 스택

```
┌─────────────────────────────────────────────┐
│  Next.js (App Router) — 모바일 웹 PWA         │
│  - 음성 입력: Web Speech API (STT) / TTS      │
│  - 동적 카드 렌더러 (JSON → React)            │
└───────────────┬─────────────────────────────┘
                │ /api/agent (서버 라우트)
┌───────────────▼─────────────────────────────┐
│  Orchestrator (OpenAI function-calling)       │
│   ├ tool: runCouncil   → tobl(cocoun) MCP     │
│   ├ tool: getActivities→ 마이리얼트립 REST     │
│   ├ tool: getRestaurants→ API Fuse·캐치테이블  │
│   ├ tool: getWeather   → API Fuse·기상청       │
│   ├ tool: getRoute     → API Fuse·카카오맵     │
│   └ seed: GenRank GET (무인증)                 │
│  EXAONE-4.5 (팀원 Mac/로컬 추론 서버)          │
│   ├ NLU: 발화 → {date,budget,mbti}            │
│   └ 페르소나 대화 카피 생성                    │
└──────────────────────────────────────────────┘
```

- **프론트**: Next.js + Tailwind. 카드 컴포넌트는 discriminated-union JSON으로 렌더.
- **EXAONE 서빙**: 팀원 Mac에서 로컬 HTTP 엔드포인트(예: llama.cpp/vLLM/Ollama-호환)로 노출 → Next 서버가 호출. **세션①에서 엔드포인트 URL 확정 = 첫 블로커.**
- **키 관리**: `.env.local` 분리, `.gitignore` 등록 (서약서 필수).

## 5. Voice / EXAONE 통합 명세 (LG U+ 트랙 필수)

1. **STT**: Web Speech API(`SpeechRecognition`, ko-KR). 실패 시 텍스트 칩 폴백.
2. **NLU (EXAONE)**: 전사 텍스트 → 구조화 JSON. 프롬프트 예:
   ```
   입력: "이번 주말 한 십오만원으로 여친이랑 놀거야"
   출력: {"date":"이번주말","budget":150000,"party":2,"relation":"커플"}
   입력: "주말에 혼자 십만원으로 놀고 싶어"
   출력: {"date":"주말","budget":100000,"party":1,"relation":"혼자"}
   ```
   → party 미언급 시 S0 선택값(mode)으로 보정.
3. **페르소나 대화 (EXAONE)**: MBTI별 말투로 council 코멘트 한국어 생성. duo는 두 사람 페르소나 대화, solo는 내 MBTI 페르소나 + 취향 페르소나의 셀프 토론 ("ENTP인 너는 새로운 거 좋아하지? 혼자 방탈출도 의외로 꿀잼ㅋㅋ").
4. **TTS**: S4 페르소나 투표 결과를 음성으로 읽어줌(`SpeechSynthesis`). → "음성 입출력 모두" 성립.

> LG U+ 심사 포인트: 음성이 **핵심 인터랙션**이어야 함 → S1·S2·S4를 음성 우선으로 설계(탭은 폴백).

## 10. 디자인 명세 (for Claude Design)

> 이 섹션은 GGUI 미사용에 따라 **인터랙션·UI 레이아웃을 직접 설계**하기 위한 핸드오프 명세입니다. 아래 제약을 지키되, 비주얼 디테일(정확한 색·여백·모션 곡선)은 디자이너 재량으로 제안해 주세요.

### 10.1 제품 성격 / 무드
- **한 줄 무드**: "혼자도 둘이도, 설레고 가벼운 주말 큐레이터"
- **무드 키워드**: 따뜻함 · 트렌디 · 장난기(ㅋㅋ 섞인 반말 카피) · K-데이트 감성 · 깔끔
- **톤 오브 보이스**: 친구처럼 반말. AI 페르소나는 MBTI별 말투 차이가 드러나게. (예: ENTP "오 이거 완전 새롭다 가보자ㅋㅋ" / ISFJ "여긴 분위기 좋고 편할 듯")
- 피해야 할 것: 광고형 리스트 느낌, 차갑고 정보 빽빽한 대시보드 느낌

### 10.2 디자인 원칙 (이 앱 고유 제약)
1. **모드 인식 (1인 / 2인)** — S0에서 solo·duo 분기. **duo**는 폰을 테이블에 놓고 둘이 함께 보며 "이제 네 차례" **턴 전환 큐**로 번갈아 조작(단일 세로 방향 유지, 양방향 회전 UI는 범위 밖). **solo**는 턴 큐 없이 1인 단독 플로우, 카피도 "너/네" 단수형. 두 모드가 같은 화면 골격을 공유하되 인원수·카피·슬롯 수만 달라지도록 설계.
2. **한 화면 = 한 결정** — 각 스텝은 질문 하나·선택 하나. 인지 부하 최소화.
3. **엄지 도달권** — 1차 액션(마이크·주요 카드·CTA)은 화면 하단 60% 안에 배치.
4. **생성형 카드** — 카드는 고정 템플릿이 아니라 **LLM JSON으로 동적 렌더**(§10.6). 매 턴 내용·개수가 달라짐을 전제로 레이아웃(가변 그리드)을 설계.
5. **음성 우선, 탭은 폴백** — 모든 입력 화면에 마이크가 주인공, 텍스트/그리드는 보조.

### 10.3 비주얼 시스템 (출발점 — 디자이너가 확정)
- **모드**: 라이트 기본 + 다크 대응(데모는 라이트로 시연)
- **컬러 방향**: 부드러운 그라데이션 배경(코럴·피치 → 라벤더 계열, "설렘"), 카드는 화이트/뉴트럴, 강조 1색(따뜻한 코럴) + 보조 강조(딥 퍼플=AI/페르소나)
- **타이포**: Pretendard(한국어 가독). 큰 질문 헤드라인 + 작은 보조 텍스트 대비 강하게
- **형태**: 큰 라운드(카드 radius 20–24px), 부드러운 그림자, 넉넉한 여백
- **아이콘/이모지**: 키워드·코스에 이모지 적극 활용(친근함). 단 과하지 않게

### 10.4 레이아웃 그리드
- 모바일 우선, 단일 컬럼, `max-width 480px` 중앙 정렬(태블릿/데모 미러링 대비)
- 안전 영역(노치·홈바) 패딩 확보, 하단 CTA는 sticky
- 카드 그리드: 키워드=2열 칩 그리드, 코스=1열 풀폭 스택(가로 스와이프 대안 가능)

### 10.5 화면별 레이아웃 해부 (S0–S6)
각 화면: **상단(컨텍스트)** / **본문(콘텐츠)** / **하단(주 액션)** 3단으로.

| 화면 | 상단 | 본문 | 하단(주 액션) | 핵심 상태 |
|---|---|---|---|---|
| **S0 모드 선택** | 로고 "사이" | "오늘 누구랑 놀까?" + [🙋 혼자]/[💑 둘이서] 큰 선택 카드. duo 선택 시 "폰을 둘 사이에 놓아주세요" 안내 | 모드 탭 / 음성 선택 | solo·duo 선택 상태 |
| **S1 음성·예산** | 진행 점(1/4) | 큰 질문 "이번 주말, 얼마로 뭐할까?" | 🎙️ 마이크(대형, 중앙 하단) + 전사 텍스트 / 예산 칩 폴백 | listening·processing·fail→칩 |
| **S2 MBTI** | 진행 점(2/4) | solo="네 MBTI는?"(1슬롯) / duo="둘의 MBTI는?"(2슬롯: 나/상대) | 음성 / 4×4 MBTI 그리드 탭 | 슬롯별 입력됨·완료(인원수만큼) |
| **S3 키워드** | 진행 점(3/4) | "끌리는 거 골라봐" + 키워드 카드 6–8개 | 다중 선택 → [다음] | 선택 0개(비활성)/n개 |
| **S4 페르소나 투표** | "AI 친구들이 토론 중 👀" | 페르소나 아바타(2–3명, 인원수+트렌드) + 코멘트 말풍선 + 투표 집계 바 | 자동 진행 + TTS 음성 | 토론 중(로딩)→결과 reveal |
| **S5 코스 카드** | solo "너를 위한 코스" / duo "너희를 위한 코스" + 💰예산 합계 미터 | 코스 카드 스택(액티비티/맛집), 각 카드에 ☀️날씨·📍거리 배지 | 카드 탭 → 송객 / [다시 추천] | 생성 중 스켈레톤→카드 |
| **S6 송객** | — | 외부 페이지(마이리얼트립/캐치테이블) | 시스템 딥링크 | 이동 전 확인 |

### 10.6 생성형 카드 JSON 스키마 (디자인이 이 형태를 렌더)
LLM이 아래 discriminated union으로 카드를 반환 → UI는 `type`별 변형 렌더:
```ts
// 키워드 카드 (S3)
{ type: "keyword", id, emoji, label, vibe /*한줄 분위기*/, hot?: boolean }

// 코스 카드 (S5)
{ type: "course", slot: "activity" | "food",
  title, image, price /*원*/, distanceText, 
  weatherFit: "indoor" | "outdoor" | "any",
  badges: string[] /*예: ["원데이클래스","분위기맛집"]*/,
  deeplink /*송객 URL*/ }

// 페르소나 투표 결과 (S4)
{ type: "vote",
  personas: [{ name /*ENTP*/, avatar, comment, pickId }],
  tally: [{ optionId, label, votes }] }
```
> 디자인은 각 `type`에 대한 **빈/로딩(스켈레톤)/에러 변형**까지 함께 제시해 주세요.

### 10.7 인터랙션 & 모션
- **턴 전환**: 화면 전환은 가로 슬라이드, 진행 점으로 위치 표시
- **마이크**: idle(은은한 펄스) → listening(파형/링 확장) → processing(스피너) 3상태
- **카드 등장**: stagger fade-up(0.05s 간격), 탭 시 살짝 눌림(scale 0.97)+햅틱
- **S4 투표**: 코멘트 말풍선 순차 등장 → 집계 바 채워지는 애니메이션 → 결과 강조
- **예산 미터(S5)**: 합계가 15만원 게이지에 차오르는 모션, 초과 시 색 경고
- `prefers-reduced-motion` 시 모션 최소화

### 10.8 상태·폴백 (반드시 디자인)
- **STT 실패** → 텍스트 입력 칩/그리드로 자연스럽게 폴백(에러처럼 보이지 않게)
- **로딩** → 각 화면 스켈레톤(특히 S4 토론, S5 카드 생성)
- **빈 결과**(예산 내 상품 없음) → "예산을 조금 늘려볼까?" 제안 CTA
- **API 에러/429** → 재시도 + 캐시된 데모 결과 표시(시연 안전장치)

### 10.9 음성 UI 어포던스 (LG U+ 트랙 가시성)
- 마이크 상태가 항상 명확(듣는 중/처리 중/말하는 중)
- 전사 텍스트 실시간 표시 → 잘못 들으면 바로 수정 가능
- TTS 재생 중엔 스피커 인디케이터 + 자막 동시 노출(duo는 둘이 함께 보므로 자막 필수, solo도 소음 환경 대비)

### 10.10 접근성·물리 제약
- 탭 타깃 ≥ 44px, 텍스트 대비 WCAG AA
- 한 손/번갈아 조작 가정 → 하단 액션 우선
- 데모는 실제 스마트폰 1대 기준(미러링) → 실기기 가독성 우선
